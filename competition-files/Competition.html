Data preparation

Select products for modeling:

# Filter products based on sales volume or other criteria
selected_products <- inventory_data %>%
  group_by(item_no) %>%
  summarise(total_sales = sum(sold)) %>%
  filter(total_sales > 1000) %>%
  pull(item_no)

Fitting model

Next Fit a Poisson regression model for each selected product:

# Function to fit Poisson regression model for a given product
fit_poisson <- function(data) {
  # Fit Poisson regression model
  poisson_model <- glm(sold ~ week, data = data, family = poisson)
  
  # Make predictions
  predicted_sales <- predict(poisson_model, type = "response")
  
  return(predicted_sales)
}

# Fit Poisson regression model and make predictions for each selected product
product_predictions <- lapply(selected_products, function(product) {
  product_data <- inventory_data %>%
    filter(item_no == product)
  
  fit_poisson(product_data)
})

In the code above, the fit_poisson() function is defined to fit a Poisson regression model using the glm() function from base R. It uses the “week” variable as the predictor for sales (“sold”) and specifies the family as Poisson.

The lapply() function is then used to apply the fit_poisson() function to each selected product, resulting in a list of predicted sales values for each product.

Model evaluation

Finally evaluate the goodness of predictions:

# Evaluate the goodness of predictions
evaluate_predictions <- function(product, prediction_data) {
  actual_data <- inventory_data %>%
    filter(item_no == product)
  
  # Calculate metrics (e.g., MAE)
  mae <- mean(abs(actual_data$sold - prediction_data))
  
  return(mae)
}

# Evaluate the goodness of predictions for each selected product
evaluation_results <- lapply(1:length(selected_products), function(i) {
  product <- selected_products[i]
  prediction_data <- product_predictions[[i]]
  
  evaluate_predictions(product, prediction_data)
})

# Calculate overall performance metric (e.g., average MAE)
overall_performance <- mean(unlist(evaluation_results))
overall_performance
## [1] 101.8706

The evaluate_predictions() function is defined to calculate the MAE (Mean Absolute Error) between the actual sales data and predicted sales values for a specific product. The lapply() function is used to apply this function to each product and store the results.

Finally, an overall performance metric (e.g., average MAE) is calculated by averaging the individual evaluation results.

The overall performance metric of 101.8706 (e.g., average MAE) provides an indication of the average absolute difference between the actual sales data and the predicted sales values across all selected products.It indicates that, on average, the predictions made by the Poisson regression model differ from the actual total sales values by approximately 101.8706 units.

Monthyly inventory prdictions in csv format

After we’re done evaluating the model we can go ahead and get monthly inventory stock predictions as predicted by our poisson model

# Create a new data frame to store the aggregated inventory needs
inventory_needs <- data.frame(
  week = numeric(),
  inventory_needed = numeric(),
  number_of_items_needed = integer(),  # Changed to integer type
  stringsAsFactors = FALSE
)

# Iterate over each selected product
for (i in 1:length(selected_products)) {
  product <- selected_products[i]
  prediction_data <- product_predictions[[i]]
  
  # Calculate monthly inventory needs
  monthly_needs <- aggregate(prediction_data, by = list(week = rep(1:12, length.out = length(prediction_data))), FUN = sum)
  
  # Add product and month information to the data frame
  monthly_needs$product <- product
  monthly_needs$month <- 1:12
  
  # Rename the aggregated column to "inventory_needed"
  names(monthly_needs)[3] <- "inventory_needed"
  
  # Convert "x" column to integer and remove decimals
  monthly_needs$number_of_items_needed <- as.integer(monthly_needs$x)
  
  # Remove the original "x" column
  monthly_needs$x <- NULL
  
  # Append the monthly inventory needs to the main data frame
  inventory_needs <- rbind(inventory_needs, monthly_needs)
}
